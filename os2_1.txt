Porovnání dvou operačních systémů
OS/2 versus Windows 95 
Zanedlouho se snad konečně na trhu objeví nový operační systém Windows 95 firmy Microsoft. Zaměřením a svými parametry se dá nejvíce srovnávat s již dlouho existujícím operačním systémem OS/2 Warp firmy IBM.

OS/2 Warp
Systém OS/2 je dodáván již delší dobu. Existuje již ve verzi 3, nazvané Warp. Jisté mouchy již byly tedy vychytány a systém je velice stabilní. Je ale chybou IBM, že tento skvělý operační systém skoro nikdo nezná. Mnoho uživatelů totiž ani netuší, že po jeho spuštění na ně nevybafne příkazová řádka, čekající na příkazy uživatelům neznámé, ale velice intuitivní grafické uživatelské prostředí. Je to spíše systém pro individualisty a nadšence, kteří jeho vlastnosti ocení. IBM nevyužila ani výhodu spočívající v tom, že dodává hardware i software a že se Windows 95 několikrát opozdily. 

Hardware
Operační systém vyžaduje počítač s proce-sorem nejméně 386 a s pamětí 4 MB. To je ale pro provoz žalostně málo a s touto pamětí se pevný disk ani na chvíli nezastaví. Hardwarové požadavky jsou tedy stejné jako u Windows 95.
IBM připravuje ještě verzi Warpu určenou pro počítače s procesorem PowerPC. Spojením tohoto hardwaru a Warpu by vznikl pravděpodobně výkonný systém.

Uživatelské rozhraní
Uživatelské prostředí Warpu je grafické a plně objektové. Workplace Shell nabízí pohodlné spouštění aplikací a příjemnou práci se soubory. K dispozici je Launch Pad pro rychlé spuštění aplikací, pro prohlížení disků a pro spuštění tisku. To, co budou mít uživatelé Windows 95, mají uživatelé Warpu již dlouho.

Architektura
OS/2 Warp se zdá stabilnější a bezpečnější než Windows 95. Warp se nemusel tolik soustřeďovat na zpětnou kompatabilitu a s jeho stavbou se mohlo začít na zelené louce. Nabízí preemptivní multitasking a multithreading. Kromě tabulky FAT je možné použít i mnohem výkonnější systém HPFS.
Warp pracuje s aplikacemi OS/2, DOS a Windows. Jenotlivé aplikace jsou navzájem chráněny a i aplikace pro Windows mohou mít svůj vlastní session. Pod Warpem se ovšem nedají provozovat aplikace pro Windows 95 ani pro Windows NT.

Aplikace
Aplikací přímo pro OS/2 Warp je žalostně málo. To je také příčina toho, proč se systém moc neujal, a protože se moc neujal, nikdo pro něj ani aplikace nevyvíjí. V poslední době se situace začíná pomalu zlepšovat, ale vývojáři se tak nepředhánějí jako při tvorbě aplikací pro Windows 95. IBM se situaci také snažila vyřešit tím, že s Warpem je dodáván BonusPak, kde je  IBM Works, Access Kit a jiné aplikace. IBM Works je balík jednoduchých kancelářských aplikací. 
Ze strachu z nedostatku aplikací je ve Warpu také možnost spouštět aplikace pro Windows 3.1. Proč ale spouštět aplikace pro Windows ve Warpu?

Ostatní
Na lokalizaci OS/2 Warpu se pracuje a ještě letos by měla být k dispozici česká verze. Bohužel přijde trochu pozdě. 
Warp zatím neumožňuje propojení počítačů do sítě peer-to-peer. Chystá se ovšem Warp Connect, který umožní uživatelům práci v této síti a také v sítích NetWare a OS/2 Lan Server. Warp je ale pro připojení k síti Internet připraven lépe než Windows 95. K dispozici je už zmiňovaný Access Kit.


Windows 95
Windows 95 jsou stále ještě ve fázi testování; k dispozici je však finální bata-verze, a tak se již nedají čekat velké změny. Zvědavé uživatele Microsoft stále napíná a několikrát odložil termín uvedení. Již dnes ale systém Windows 95 viděla spousta uživatelů.

Hardware
Tak jako OS/2 Warp jsou i Windows 95 připraveny pro počítače s procesorem 386 a lepším. Stejně jako u Warpu se jako potřebná kapacita paměti RAM uvádějí 4 MB. Windows 95 s touto pamětí sice “chodí”, ale operační systém by měl “běhat”. Doporučovaných 8 MB se dá tedy jedině doporučit.

Uživatelské rozhraní
Windows značně změnily svou tvář a dnes se Windows 95 více podobají OS/2 Warpu než svému předchůdci OS Windows 3.1. Spousta novinek již byla použita v jiných systémech, Microsoft se jistě poučil u OS počítačů Apple, u OS/2 Warpu ale i u různých utilit pro Windows 3.1. Prostředí Windows 95 je objektové, podporována je technika drag-and-drop, nový je odpadkový koš, ikona start a taskbar.
V systému se již nepoužívá zvláštní správce programů a správce souborů, ale jejich funkce je sjednocena. Foldery (pořadače) mohou být vícenásobně vnořeny.
Při práci potěší i animované ikony a podpora zvukové karty. Podporována jsou multimédia. Většinu toho však zvládá i OS Warp.

Architektura
Windows 95 pracují s názvy souborů dlouhými až 255 znaků, ale používá se stále tabulka FAT, byť rozšířená. Windows 95 umožňují kompresi disku. Používán je multitasking a multithreading; ten je však ve Warpu, ale i v UNIXu už dávno. Je tedy možný běh několika aplikací najednou, ale pouze aplikací napsaných pro Windows 95. Staré 16bitové aplikace poběží jako pod Windows 3.1. 
Nové aplikace budou také navzájem lépe chráněny a budou mít svůj vlastní paměťový prostor. 16bitové aplikace poběží všechny v jedné paměťové oblasti, ale při pádu by s sebou alespoň neměly strhnout celý systém a 32bitové aplikace.
Skalní přívržence Nortonu pro DOS jistě potěší, že z něho budou moci spouštět i windowsovské aplikace.

Aplikace
Windows 95 jsou ve výhodě v tom, že výrobci softwaru se předhánění v tvorbě aplikací pro ně. Verze pro Windows 95 mají firmy již hotové, pracují na nich nebo je alespoň začnou připravovat. To u OS/2 Warpu neplatí a aplikací je nedostatek. S operačním systémem se také dodává řada nástrojů a aplikací, které se dříve musely dokupovat. Některé z nich jsou plnohodnotné a uživatelům mohou plně postačovat. Pod Windows 95 je možné spouštět aplikace pro Windows 95, pro Windows 3.1 a aplikace pro DOS.

Ostatní
Česká verze tohoto systému by měla být k dispozici krátce po uvedení verze americké. Část systému je již přeložena a poslední beta-verze je již kromě chybových hlášení a nápovědy prakticky celá česká.
Windows 95 pracují navzájem v síti peer-to-peer a sítím obecně je věnována velká pozornost.

Microsoft už dávno není micro
Firmu Microsoft zná pravděpodobně každý čtenář. Hodně se o ní mluví a ještě bude mluvit po uvedení Windows 95. Někdo ji a její produkty má rád, jiný ne. Ignorovat ji však nemůže skoro nikdo, protože v některých oblastech určuje vývoj a ovlivňuje prakticky celý svět informačních technologií. Pro čtenáře snad bude zajímavé přečíst si něco z historie a současnosti této firmy.
Firma Microsoft byla založena 4. dubna 1975 Williamam H. Gatesem a Paulem G. Allenem. Zakladatelé firmy se znali z vysoké školy. Z malé firmičky o třech lidech je dnes gigant s obratem 4,6 mld. USD (fiskální rok 1994) a čistým příjmem 1,15 mld. USD. V roce 1985 měla firma již 1001 zaměstnanců a v současnosti jich má již přes 16 000. Microsoft má pobočky v 49 zemích, jeho produkty se vyvíjejí v 30 jazycích a prodávají se ve více než 100 zemích. Za dobu své existence firma spolkla řadu menších firem a přetáhla k sobě mnoho odborníků z jiných společností.
Pro budoucnost firmy byl pravděpodobně rozhodující rok 1981, kdy vytvořila systém MS-DOS pro počítače společnosti IBM. Jde o dnes nejrozšířenější operační systém pro počítače založené na procesoru Intel. Je naistalován asi na 140 milionech osobních počítačů, což je asi 90 % ze všech. Grafickou nadstavbu OS Windows používá dnes legálně asi 60 milionů uživatelů.
Mnoho hardwarových firem dodává svoje počítače s již předinstalovanými produkty této firmy. Na českém trhu jsou dostupné všechny programy Microsoftu v anglickém jazyce a mnoho jich bylo lokalizováno. Proto se produkty této firmy prodávají dobře i u nás.
Své postavení na trhu si musel Microsoft samozřejmě vybojovat a často riskovat a má za sebou i několik neúspěšných projektů. Firma se ale vždy rychle přeorientovala na perspektivní věci. I dnes se pouští do všeho nového. V poslední době je to třeba informační dálnice nebo multimediální aplikace. Než se ostatní rozkoukají, Microsoft již obsadí trh s touto rychle se rozvíjející oblastí a ostatní pak budou poukazovat na jeho monopolní postavení. Pouští se i do projektů pracujících na počítačovém rozpoznávání hlasu, na trojrozměrném grafickém prostředí nebo do bio-technologických projektů.
Microsoft se nesoustřeďuje pouze na počítače IBM PC-kompatibilní, ale vyvíjí software i pro počítače Apple. Velice dobře se prodává Excel a Word pro počítače Apple, což je důkaz, že se aplikace neprodávají jen kvůli operačnímu systému Windows. Operační systém Windows NT je použitelný stále na více platformách. Microsoft se zaměřuje na všechny typy uživatelů a vyvíjí operační systémy (Windows, Windows NT), obchodní systémy (Windows NT Server, SQL Server atd.), kancelářské aplikace (MS Excel, MS Word atd.), vývojové nástroje (MS Access, Visual Basic, Visual C++, MS Fortran, Microsoft Test), aplikace pro domácí uživatele (MS Encarta, MS Cinemania, MS Bookshelf), hry (MS Golf, MS Space Simulator, Complete NBA, Basketball) atd. Známé jsou také hardwarové výrobky Microsoftu - ergonomické myši a klávesnice. Microsoft se věnuje i tisku publikací (Microsoft Press).
Ti, kdo nemají Microsoft rádi, mají samozřejmě možnost používat produkty konkurenčních firem. K největším patří firma IBM se svým operačním systémem PC DOS a OS/2 Warp a nově je firma posílena i produkty Lotusu. PC DOS je v několika ohledech lepší než MS-DOS a OS/2 Warp již dávno nabízel většinu toho, co bude mít až OS Windows 95 firmy Microsoft. Bohužel OS/2 Warp se u nás příliš nerozšířil, možná také kvůli tomu, že česká verze se teprve připravuje. K silným soupeřům Microsoftu patří firma Novell se zakoupeným DR DOSem (náhradou MS-DOSu), s produkty WordPerfect a Quattro Pro (náhrada Wordu a Excelu) a hlavně se svým síťovým řešením. Novell DOS je také v mnoha ohledech lepší než MS-DOS. DR-DOS vyvíjela společnost Digital Research, která měla v roce 1981 vytvořit operační systém pro firmu IBM; k realizaci však nedošlo.

Klíčové události v historii firmy Microsoft
1975 Paul Allen a Bill Gates inspirováni článkem o minipočítači Altair 8800 začínají vyvíjet BASIC pro MITS Altair. Hotový programovací jazyk prodávají prvnímu zákazníkovi a BASIC se stává prvním počítačovým jazykem pro osobní počítače.
1976 Zlepšený a rozšířený BA-SIC se prodává dalším zákazníkům. Paul Allen a Bill Gates zakládají partnership (právní forma podniku v USA).
1977 Microsoft uskutečňuje vývoj druhého programovacího jazyka pro osobní počítače, totiž jazyka FORTRAN.
1978 Microsoft začíná prodávat programovací jazyk COBOL 80 pro počítače založené na procesorech 8080, Z-80 a 8085.
1979 Microsoft uvádí BASIC pro počítače s procesorem 8086. Jde o první verzi jazyka pro 16bitové počítače. Microsoft se rozrůstá a expanduje i do Evropy. 
1980 Microsoft začíná úspěšně dodávat na trh SoftCard system. Jde o hardwarové a softwarové rozšíření pro majitele počítačů Apple II, jež zajišťuje to, aby aplikace pro operační systém CP/M mohly být provozovány na počítačích Apple.
1981 Microsoft se mění v Microsoft, Inc. (jiná právní forma podobná naší a.s.). Bill Gates se stává prezidentem a předsedou společnosti a Paul Allen výkonným viceprezidentem. Rozhodující pro budoucnost Microsoftu je uvedení osobního počítače firmy IBM s operačním systémem Microsoftu MS-DOSem 1.0 a s programy BASIC, COBOL, Pascal aj.
1982 Microsoft zřizuje svoji první pobočku ve Velké Británii - Microsoft, Ltd. Dále představuje svůj produkt GW-BASIC, založený na standardu MS BASIC a rozšířený např. o podporu grafiky. Microsoft také ohlašuje nové logo firmy a nový způsob balení. Připraven je první produkt pro koncové uživatele - Multiplan Electronic Worksheet a dokončeny jsou práce na produktu MILAN (Microsoft Local Area Network).
1983 Microsoft představuje několik 16bitových verzí svých programovacích jazyků pro počítače s procesorem 8086, 8088 a s operačním systémem MS-DOS.  Jde o interpreter jazyka BASIC, překladač jazyka BASIC, Pascal, FORTRAN, COBOL a o jazyk C. Operační systém MS-DOS již existuje ve verzi 3.0. Microsoft se vrhl i na oblast hardwaru a vyrábí polohovací zařízení nazvané Microsoft Mouse. Vytvořen je textový procesor Word pro MS-DOS. Microsoft také oznamuje na COMDEXu grafickou nadstavbu operačního systému MS-DOS - MS Windows. Prvním pokusem je Interface Manager.
1984 Microsoft vyvíjí software i pro počítače Apple Macintosh. Mezi partnery OEM a nezávislé softwarové prodejce rozšiřuje The Microsoft Windows Software Development Toolkit. Microsoft dále dodává Project 1.0 pro MS-DOS. IBM přichází s počítačem IBM PC AT s operačním systémem MS-DOS 3.0 nebo 3.1 a s XENIXem od firmy Microsoft.
1985 Microsoft zakládá první továrnu mimo území USA, a to v Irsku. Připraven je Excel 1.0 pro počítače Macintosh.
1986 Microsoft se stává akciovou společností.
1987 Microsoft a IBM ohlašují operační systém OS/2 a OS/2 Lan Manager. Jde o první výsledky dohody mezi IBM a Microsoftem (Join Development Agreement). Microsoft také ohlašuje novou verzi Windows, a sice pod označením 2.0. Microsoft získává firmu Forethougt, Inc., která se zabývala vývojem produktu PowerPoint, a ještě v tomto roce přichází s PowerPointem 1.0 pro Macintosh. K dalším výrobkům patří první aplikace na disku CD-ROM, tedy encyklopedie Bookshelf, a dále Excel pro operační systém Windows. 
1988 Microsoft se stává největším prodejcem softwaru. V tomto roce společně s firmou Ashton-Tate připravuje Microsoft SQL Server 1.00 pro operační systém OS/2; tento server je založen na relačním databázovém systému společnosti Sybase, Inc. Uvádí také operační systém MS-DOS 4.0 a OS/2 1.1 s Presentation Managerem.
1989 Microsoft zřizuje divizi, která se zabývá multimédii. Na disku CD-ROM je dodáván balík programů nazvaný Microsoft Office a určený pro počítače Macintosh.
1990 vzniká Russian MS-DOS 4.01, první produkt lokalizovaný pro sovětský trh. Tato verze MS-DOSu přináší možnost různých jazykových mutací. Microsoft ohlašuje Windows 3.0 a Bill Gates pod heslem “Information at Your Fingertips” (Informace na dosah ruky) nastiňuje na COMDEXu svoji vizi, jež se týká využití počítačů v budoucnosti. Ohlášena je nová verze MS Excelu 3.0 pro systémy Windows, OS/2 a Macintosh.
1991 Microsoft ohlašuje vývojový nástroj Visual Basic. Operační systém MS Windows 3.0 je k dispozici ve 12 jazycích v 24 zemích světa. Ohlášeny jsou nové možnosti operačního systému MS-DOS 5.0 a SQL Serveru 4.2. Microsoft a IBM končí spolupráci. IBM se věnuje systému OS/2 a Microsoft systému Windows.
1992 Nová verze operačního systému Windows 3.10 se stává velice úspěšnou. Firma Fox Software splývá s Microsoftem, a tak tato firma získává databázi FoxPro. K vlastním novinkám patři Visual Basic zpracovaný ve standardní a profesionální verzi a určený pro operační systém MS-DOS a Visual Basic 2.0 pro Windows, prodává se MS ODBC Software Development Kit, multimediální aplikace MS Musical Instrument, ohlášena je databáze Access a beta-verze OS Windows NT. Pro nás je zajímavé, že v listopadu roku 1992 bylo otevřeno zastoupení Microsoftu v Praze.
1993 Tento rok je na novinky bohatý. Na trh je na disku CD-ROM uvedena verze MS Office, dále Visual Control Pack, nový vývojový nástroj Visual C++ ve standardní a profesionální verzi, jazyk COBOL, multimediální encyklopedie Encarta a další multimediální tituly, nová verze MS-DOSu s označením 6.0 a následně 6.2, MS Office s databází Access, myš Microsoft Mouse 2.0, MS Word 6.0 pro DOS, Windows NT a Windows NT Advanced Server a Windows 3.11. Word se stává nejoblíbenějším textovým editorem. Přichází také první lokalizovaný produkt - česká verze operačního systému Windows 3.1 - a později také Word 2.0, Excel 4.0 a Works 2.0 pro Windows.
1994 Dlouho očekávaná nová verze operačního systému Windows 95 s kódovým označením Chicago se dostává do fáze beta-testování. MS přichází se skupinou programů s názvem Microsoft BackOffice. U nás jsou lokalizovány produkty Word 6.0, Excel 5.0, Windows pro pracovní skupiny a potom i celý balík MS Office Standard 4.2 CZ.
1995 V tomto roce byly představeny tyto dvě nové verze: SQL Server 6.0 a Windows NT 3.51. Předveden byl i vývojový nástroj Visual FoxPro. Neustále se čeká na uvedení OS Windows 95. V současnosti je již ve finální beta-verzi, a to i v českém jazyce.

Detektivní příběh z roku 2004
Budoucnost podle Billa Gatese
Seattle, rok 2004. V ponurém nočním přístavu si ošuntělý zloděj odnáší z lodi svůj lup. Slyší ho muž v dlouhém kabátě a s pistolí v ruce. Vystřelí po něm, ale zloději se podaří utéci. Běží přístavem a potom se skryje za hromadu starých rybářských sítí. Když už si myslí, že unikl, zezadu k němu přijde muž v kabátě a střelí ho téměř nehlučnou pistolí do hlavy.
Příští den ráno si ve městě muž a žena (policisti) kupují u pouličního stánku kávu. Dojde k placení. Na displeji kasy je vidět účtenka s celkovou sumou k zaplacení. Stejné údaje má i žena na zařízení s displejem o velikosti “digitální hry” (řekněme PDA - Personal Digital Assistant), které odněkud vytáhla. Něco na něm zmáčkne a na kase prodavače se objeví “zaplaceno”.
Oba pak nastupují do auta, ve kterém mají plochou obrazovku s úhlopříčkou asi 30 cm. Na obrazovce se objeví muž, s nímž si povídají, a dostávají pokyny. Dvojice vidí i záběry kamery z místa činu noční vraždy, textové informace, fotografie a mapu města s vyznačeným místem činu.
Mezitím se chlapec Jackson doma připravuje do školy. Pomocí svého “počítače” si prohlíží informace o umění a architektuře Jižní Ameriky. Zajímá ho umění z předkolumbovské doby. Vidět jsou sošky, nádoby, šperky, fotografie míst a textové informace. Informace získává chvíli z knihovny, z univerzity, chvíli z muzea v Mexiku. Nakonec si vyhledá společnost, která prodává repliky těchto uměleckých předmětů. Zjistí si cenu zajímavé masky, adresu firmy a na mapě si prohlédne ulici s obchodem.
Jacksonova matka se zatím dívá na televizi. Pomocí dálkového ovládání si postupně vybírá z nabídky to, co ji zajímá. Často sledované pořady má znázorněny ikonou a v kteroukoli dobu si může vybrat film, pustit si zprávy, písničky, sportovní události, zjistit, jaké bude počasí, sledovat svoji oblíbenou show, prohlídnout si poštu nebo si může popovídat s někým známým. Přichází za ní Jackson a prosí ji, aby šli vybranou masku koupit. Nutně ji potřebuje do projektu, který si chystá do školy.
V prodejně replik a uměleckých předmětů je spousta zajímavých věcí. Je tu i obraz, který může měnit své téma. Matce a chlapci se věnuje prodavač a na měnícím se “obraze” si prohlížejí předměty, které chlapce zajímají. Najdou i masku, o kterou má Jackson zájem.
Na první pohled podezřelý muž vybaluje zboží ve skladu prodejny. Pípá mu jeho PDA. Displej muži oznamuje, aby šel k telefonu. Přichází k prvnímu volnému počítači a ze záznamu vidí volajícího. Volá ho muž v kabátě, vrah z přístavu ...
Prodavač zjišťuje, že vyhledaná maska je prodaná. Nové zboží přivezou až další lodí. Zklamaný chlapec s matkou se chystají k odchodu. Prodavač odchází něco udělat do skladu a nalézá masku. Právě ji vybaloval podezřelý muž, ale musel ji opustit kvůli telefonu. Radostně ji přináší zákazníkům, zabalí ji a naúčtuje. Jackson na svém PDA zjišťuje, že potřebný obnos nemá. Poprosí matku a ta mu půjčuje. Na displeji zadá částku a ikona bankovky přeběhne přes displej. Přístroje jen zapípají, transakce je u konce a zákazníci odcházejí. Podezřelý muž dotelefonoval a hledá masku. Ptá se prodavače a po zjištění, že je maska prodána, je zděšený.
Ve škole panuje značný ruch. Děti si hrají se svými přístroji velikosti tabulky, která se společně s křídou dříve používala ve škole na psaní. Jackson na tomto přístroji na poslední chvíli dokončuje svůj domácí úkol. Vchází učitelka a prosí o klid. K tabuli jde Jackson. Spouští svoji prezentaci předkolumbovské kultury, která by se v PowerPointu určitě nevytvořila. Na velikém displeji, který je ve škole místo tabule, se objevují předkolumbovské postavičky, sošky, animace trojrozměrných předmětů, hrnce a jiné umělecké předměty, obrázky, mapy a vše je podbarveno hudbou. Jackson má k prezentaci svůj výklad. Děti i učitelka jsou nadšeni. Domácí úkol se povedl.
Jackson vychází ze školy. Z auta ho sleduje muž z obchodu, který tolik postrádá Jacksonovu masku. Spolu s ním je v autě vrah. Chlapec je v nebezpečí. Policisté zatím pátrají v okolí místa činu, pátrají i na lodi. Dostávají zprávu od FBI na svůj PDA a získávají první stopy.
V místní nemocnici sestra a doktoři sledují obrazovku. Je na ní záběr ze sanitky současně se záběrem hovořícího doktora, dále graf činnosti srdce převáženého pacienta a obrázek vnitřku těla. Jackson je zraněn, ale naštěstí to není nic vážného. Zakrátko už k němu do nemocničního pokoje může na návštěvu matka. Přicházejí i vyšetřovatelé vraždy a ptají se Jacksona, co se mu přihodilo. Ukazuje jim i svoji masku.
Vše se začíná vyjasňovat a policisté si jdou prohlédnout i obchod s reprodukcemi. Podezřelého muže se policisté ptají na různé věci a také na prodej v minulém týdnu. Muž spouští obrazovku a zobrazuje na ní prodávané předměty a obchodní údaje. Rozhovor se zaznamenává do přístroje policistů, a to tak, že mluvené slovo se převádí automaticky na text; přístroj rozpoznává i to, kdo zrovna mluví. Podezřelý však nechce sdělovat všechny informace a poukazuje na ochranu zákazníků.
Policisté odcházejí, ale jsou přesvědčeni, že muž je v něčem namočen. Podezření se jim potvrzuje i v autě, do kterého zatím nastoupili a kde se jim na displeji objevuje nová zpráva s popisem pachatelů a s nákresem jejich vzhledu, který vytvořil Jackson.
V noci někdo tiše vchází do domu, kde bydlí Jackson s matkou. Je to muž z obchodu a bere z domu Jacksonovu masku. Bezpečnostní systém však nevítaného návštěvníka odhaluje a spouští alarm. Přibíhá matka a muž ji zastavuje s pistolí v ruce. Hrozí jí, že ji zastřelí, pokud nezůstane stát. Matka volá chlapce a utíká. Muž ji střelí. Příběh končí.
Ale ne, to by nebyl příliš veselý konec. Do děje naštěstí vstupuje Bill Gates a vybírá ikonu alternativního konce - happy endu. 
Matka neutíká, ale vytáhne na muže pistoli. Není to totiž Jacksonova matka, ale převlečená policistka. Objevují se i další policisté a muže zatýkají. Vyšetřovatel pouští masku na zem a ta se rozbíjí. Pod tmavou slupkou se objevuje zlato...
Tento příběh pustil ve filmové podobě a komentoval na COMDEXu Bill Gates. Chtěl tak názorně a zábavně ukázat, jak si představuje využití počítačů v budoucnosti. Počítače by se měly snadno ovládat pomocí výkonných grafických objektových operačních systémů. Umožňovaly by získávat informace z různých zdrojů, telefonovat, přenášet velké objemy dat a hlavně by měly být všude dostupné. Pomocí osobních digitálních asistentů by se platilo a komunikovalo. Film byl po technické stránce dokonalý a zobrazované technické přístroje vypadaly realisticky.
Bude zajímavé sledovat, zda se vize tohoto muže mění v praxi, nebo zda vývoj počítačů a telekomunikací přecenil, popřípadě podcenil.
Rok 2004 je tu v podstatě za chvíli, ale dá se předpokládat, že tak, jak se zvyšuje výkon procesorů a počítačů dnes, bude se měnit i v budoucnu.  V roce 1981 měl procesor 8088 výkon 0,33 MIPS, procesor 286 z roku 1984 1,5 MIPS, procesor 386 z roku 1990 již 6 MIPS a procesor 486 z roku 1990 20 MIPS. Dnes nejnovější procesor firmy Intel - Pentium - zvládá již 100 MIPS a vývoj se samozřejmě nezastavil. Jak je vidět, růst výkonu procesorů není lineární, ale exponenciální.
Pokud budou počítače roku 2004 skutečně natolik výkonné, aby zvládly předvedená kouzla, musí se ještě počítat s tím, že ne všichni lidé budou ochotni výpočetní techniku akceptovat.

Plug and Play a Windows 95
Plug and Play (“zapoj a spusť”) je systémová architektura schopná indentifikovat a konfigurovat hardware a operační systém osobních počítačů, a v důsledku toho usnadňuje instalaci a odpojení hardwarového zařízení. Tento systém se skládá ze tří komponentů: z operačního systému podporujícího Plug and Play, z BIOSu s podporou Plug and Play a z hardwarového zařízení kompatibilního s touto architekturou.
Na vytvoření specifikace Plug and Play se podílela řada firem spolupracujících s asociací Plug and Play. Tato technologie má odstranit někdy složité nastavování přepínačů a jumperů, nahrávání různých ovladačů, nastavování DMA a měla by řešit jinak možné konflikty s IRQ. Už na COMDEXu v roce 1993 bylo k dispozici 18 zařízení podporujících Plug and Play. Dnes je těchto zařízení, která vyrábí desítky firem, k dispozici  celá řada. Používá se i logo - např. “Designed for Microsoft Windows 95” pro výrobky vyhovující specifikaci Plug and Play určené pro OS Windows 95.
Ne vždy jsou však všechny tři komponenty k dispozici. Pokud operační systém, zařízení ani BIOS počítače nejsou kompatibilní se specifikací Plug and Play, vše se musí nastavit ručně (přepínače na kartě, ovladače). Za situace, kdy je operační systém připraven na technologii Plug and Play, avšak zařízení ne, pomáhá tento systém při instalaci hardwaru. Pomáhá detektovat, identifikovat a konfigurovat instalované zařízení. Pokud je operační systém, zařízení i BIOS počítače kompatibilní se specifikací Plug and Play, instalace by měla proběhnout hladce.
S Windows 95 získáte operační systém podporující tuto technologii. Zařízení mohou být připojena přes mnoho typů sběrnic (ISA, VL, PCI, EISA) a konektorů (PCMCIA, SCSI, IDE, sériové a paralelní porty a monitory). Připojená zařízení se pak mohou automaticky konfigurovat. Podle Microsoftu je možno do počítače s OS Windows 95 snadno instalovat i starší hardware, nepřipravený na tuto technologii, a to i do počítače bez BIOSu obsahujícího Plug and Play. Lze automaticky nastavovat monitory, tiskárny, videokarty a zvukové karty, adaptéry SCSI, modemy a mechaniky CD-ROM. 
Uživatelé mají možnost připojovat a odpojovat zařízení PCMCIA bez vypnutí počítače. Počítač tyto změny automaticky zaznamená a počítá s nimi. Notebooky lze vyjmout z rozšiřující stanice (docking station) za provozu.
Ve Windows 95 slouží ke snadné instalaci tyto komponenty: Configuration Manager, Hardware Tree and Registry, Bus and Port Enumerator a Setup and Device Installer.
–ptr

Windows 95 stále předmětem sporu
Kdo je proti?
Jen málokterý softwarový produkt je mezi představiteli americké justice tak populární jako Windows 95. Vzhledem k dominantnímu postavení Microsoftu jsou nové Windows trnem v oku všem konkurentům, kteří v tomto operačním systému právem vidí ohrožení své pozice na trhu. K softwarovým firmám se v první polovině tohoto roku připojily společnosti poskytující on-line informační služby: America Online, CompuServe a Prodigy. Tyto společnosti se obávají chystané sítě Microsoft Network (MSN), jejíž začlenění do Windows 95 údajně porušuje pravidla svobodné soutěže. Spory došly tak daleko, až se dokonce v kuloárech začalo zpochybňovat kalendářní datum v názvu Windows 95 - do hry se totiž zapojil i americký antimonopolní úřad, který poslední dobou rád hraje roli jakéhosi softwarového četníka. V posledních měsících dokonce vztahy Microsoftu a tohoto úřadu připomínají hrátky Toma a Jerryho.

Partneři OEM
Válku proti Microsoftu zahájily konkurenční firmy ve spojení s antimonopolním úřadem hned na dvou frontách. Zatímco výrobci osobních počítačů si stěžovali na údajně diskriminační přístup Microsoftu v oblasti licenční politiky, poskytovatelé on-line služeb obvinili Microsoft z toho, že získá na trhu příliš dominantní postavení a ohrozí tak principy svobodné soutěže. Větší šance na úspěch se všeobecně přikládaly výrobcům počítačů, a jak události posledních dvou měsíců ukázaly, tento odhad byl správný. Microsoft počátkem července údajně změnil podmínky poskytování licencí výrobcům OEM a získal tak na svou stranu většinu největších počítačových firem. V polovině července údajně na seznamu OEM-partnerů chybělo ještě zhruba šest firem, přičemž Microsoft odmítl poskytnout podrobnosti se zdůvodněním, že jednání dosud probíhají. Podle informací časopisu Information Week byly mezi nimi například společnosti Hewlett-Packard a IBM.
17. července zahájil Microsoft distribuci finální verze Windows 95 všem OEM-partnerům. Podle zdrojů blízkých firmě tak učinil přesto, že výsledky testů poslední beta-verze poukazovaly na množství přetrvávajících chyb. Podle Redmondu stále existuje zhruba 200 aplikací, které pod Windows 95 nefungují. Jak se zdá, datum 24. srpna se stalo metou, pro jejíž dosažení je Microsoft ochoten mnohé obětovat.

Válka on-line
Sporný klient pro MSN ovšem stále zůstává součástí systému. Tato informace spolu s prohlášením, že MSN bude okamžitě od 24. srpna nabízet přímý přístup na Internet, vzbudila mezi provozovateli on-line služeb hluboké zděšení. Jediné kliknutí myší bude uživatelům Windows 95 stačit na to, aby získali WWW-brow-ser Microsoftu nazvaný Internet Explorer. Microsoft se sice snažil nedráždit své konkurenty nad únosnou míru a nezahrnul Internet Explorer do standardního balíku Windows 95, ovšem uživatelé jej mohou obdržet okamžitě poté, co se přihlásí do sítě Microsoft Network. Přes původně skeptické odhady to tedy vypadá, že Gatesova konkurence Internetu bude provozuschopná okamžitě po 24. srpnu.
Úsilí antimonopolního úřadu zabránit startu MSN zatím vyznívá naprázdno. První jednání soudu, které bylo ohlášeno na 24. červenec a které mělo projednávat právě obvinění z ohrožování svobodné soutěže, bylo totiž pouhé dva dny předtím odvoláno. Microsoft, vědom si své síly, neposkytl ministerstvu spravedlnosti požadované údaje, a naopak obvinil antitrustový úřad z podjatosti. Zřejmě dobře věděl, co dělá - v den, kdy mělo první jednání probíhat, federální úřady od vyšetřování celé záležitosti upustily.
Advokáti z kruhů blízkých americkému obchodnímu soudu se dosud neshodli, zda antimonopolní úřad vskutku pustil causu Microsoft k vodě, nebo zda si jen hraje na schovávanou. Jisté je, že do 24. srpna mnoho času nezbývá. Společnosti America Online, CompuServe a Prodigy, které jsou budoucí sítí MSN ohroženy nejvíce, tedy musí jednat rychle. Proto ani příliš nepřekvapila informace, která z politických kruhů prosákla koncem července - zástupci těchto společností totiž lobbovali v obou komorách Kongresu. Stranou zájmu “on-linistů” nezůstali ani Newt Gingrich a Robert Dole.

Recenze
Forehelp
Forehelp je původním produktem firmy ForeFront, který začala pod svojí hlavičkou šířit firma Borland. Tento nástroj by měl zjednodušit obtížné a pomalé vytváření nápovědy k vašim programům pro Windows.

Program pro Windows, který by nezahrnoval kontextově citlivou nápovědu, si ani nedokážeme představit. Vytváření nápovědy pro Windows je dlouhá, nezáživná a obtížná činnost. Psaní nápovědy bylo mnohdy bezmála větším dobrodružstvím, než psát program.
Doposud bylo nutné nápovědu psát v nějakém textovém editoru, který umožňoval text exportovat do formátu RTF (Rich text format). K tomuto souboru se dopíše v běžném ASCII-editoru jakýsi “Project file” (seznam RTF-souborů a dalších důležitých informací o nápovědě) a takto připravená nápověda se “přeloží” Help Compilerem od Microsoftu. Před několika lety jsem zkoušel vytvořit RTF-soubor z několika textových editorů, ale Help Compiler bral pouze RTF vytvořený Wordem (možná že dnešní editory jsou na tom lépe). Ve zdrojovém textu nápovědy se podle speciálních pravidel definovaly jednotlivé stránky nápovědy, odkazy mezi jednotlivými stránkami, rejstřík apod.

Instalace a dokumentace
Instalace se provádí ze dvou disket a na můj vkus trvala poměrně dlouho (cca 10 minut), byla  však bezproblémová. Hardwarové nároky na produkt jsou minimální (PC 386, 4 MB paměti, cca 5 MB místa na pevném disku). Kromě Windows 3.1 je potřeba vlastnit Microsoft Help Compiler (dodává se s mnohými produkty, ale údajně je též zdarma k dispozici u Microsoftu).
K ForeHelpu se dodává jedna středně tlustá kniha (246 stran) klasického borlandovského formátu. Obálka je sice ve stylu borlandovských manuálů, ale uvnitř se tradičně přehledný styl borlandovských manuálů nevyskytuje. Přesto je dokumentace velmi podrobná a na dobré úrovni.
Tento produkt může teoreticky spolupracovat se všemi programovacími jazyky pro Windows, ale nejvíce se asi využije s překladači C/C++ a Pascalu (resp. Delphi). 

Psaní nápovědy
Pro okno s nápovědou je především potřebné specifikovat hlavní parametry (např. copyright, pozici na obrazovce apod.). 
Nejdůležitější v nápovědě jsou různé odkazy. Existují v podstatě dva typy odkazů - nová stránka se zobrazí “přes celé okno” nebo nová stránka se objeví pouze jako modální “okénko” pod položkou (tento způsob se hodí např. pro stručné vysvětlivky k pojmu). V editoru stačí jen vybrat slovo, typ odkazu a stránku, která se má zobrazit.
Při psaní nápovědy můžeme samozřejmě do textu vřazovat různá makra, která mohou ovlivnit vzhled a chování nápovědy (těchto maker se ale příliš často nepoužívá). Pomocí těchto maker je možné ovlivňovat menu, tisknout, spouštět programy apod. Tato makra je možné přehledně vybírat ze seznamu.
Pokud chcete používat tlačítek na listování nápovědou vpřed a vzad, umožňuje vám ForeHelp nadefinovat si posloupnosti stránek, kterými se potom bude procházet. Můžeme nadefinovat i více posloupností. 
Aby byl proces vytváření nápovědy doveden až do konce, máte možnost přepnout ForeHelp do testovacího modu, v němž se ForeHelp chová skoro jako windowsový prohlížeč nápovědy. Dále je možné z prostředí ForeHelp spustit překladač nápovědy (Microsoft Help Compiler). Nakonec je též možné z ForeHelpu spustit WinHelp s hotovou nápovědou.
Aby mohl váš  program používat kontextově citlivou nápovědu, potřebuje znát odkazy na čísla stránek nápovědy. ForeHelp umožňuje generovat soubor (.H) s konstantami, které označují jednotlivé stránky nápovědy.

Interní editor
Interní editor, ve kterém vlastní text nápovědy píšete, je vzdáleně podobný Wordu. Jedná se o poměrně komfortní editor, který má velký arzenál funkcí, pomocí kterých vám umožňuje provádět běžné editovací a formátovací příkazy.
V textu nápovědy můžete samozřejmě vyhledávat a zaměňovat text. Práce s clipboardem je samozřejmostí, ale přenášení textu metodou drag--and-drop bohužel není podporováno. Editor také obsahuje jednoúrovňovou funkci Undo (a Redo). Do textu je možné vkládat obrázky, odkazy, tabulky apod. Text můžete psát libovolnými fonty; tyto fonty mohou být tučné, v kurzivě nebo podtržené. Též je možné používat různé barvy textu. Pro snadné formátování textu je možné pracovat se styly textu! Pro odstavce si můžete nastavit zarovnávání (doleva, doprava nebo doprostřed), odsazení řádků, mezery před odstavcem a za ním, tabulátory apod. 
Z dalších nástrojů, které našinec bohužel asi nevyužije, bych jmenoval SpellChecker a Thesaurus - obojí v anglické verzi. Pokud ale píšete nápovědu v angličtině, tyto nástroje se hodí. 

Závěr
Tento produkt je pro náš trh - postupně se přeorientovávající na svět Windows - velmi zajímavý. Jestiže programujete pro Windows a potřebujete udělat nápovědu, ForeHelp si opatřete. I kdybyste jej využili jen jednou, stojí to za to!
Programu se nedá téměř nic vytknout. Jedinou výtku, kterou bych uvedl, je poměrně nevhodná nástrojová lišta. Například mi v ní chybí “vyklápěcí” seznam fontů a stylů (takový, jaký je např. u Wordu nebo WordPerfectu).  V editoru mi také chybí lokální menu, které by se objevilo po kliknutí pravým tlačítkem.

WinText
WinText nemá se svým dosovským kolegou mnoho společného a možnosti tohoto textového editoru jsou srovnatelné s jinými profesionálními textovými editory, kterým se dosti podobá.
WinText umožňuje formátovat text obvyklým způsobem. Pro text je možné nastavovat font, velikost a typ fontu (podtržení, kurziva, tučné), barvy a další případné efekty. Pro odstavec můžeme nastavit tabulátory, stínování, odsazení, počet sloupců, zarovnávání a řádkování a ještě další parametry. Pro snazší formátování textu je možné používat styly odstavců.
WinText má též další nástroje pro zpracování textů. Prvním z těchto nástrojů je korektor překlepů – ve standardní dodávce korektor podporuje češtinu a slovenštinu. Je ale možné dokoupit podporu němčiny a angličtiny (britské i americké). Dalším doplňkem je rozdělování slov, které používá stejné slovníky jako korektor. K dispozici je také synonymický slovník - ale jen pro češtinu a slovenštinu. Poslední nástroj z této řady je překladový slo  vník. Ke slovníku je možné dokoupit slovníky německé a anglické (obousměrné).
Službou, která je důležitá pro firmy, může být vytváření hromadné korespondence. WinText umožňuje označit v textu místa, kam se budou při tisku vkládat údaje z databáze.
WinText již ve starších verzích podporoval práci se šablonami. Pomocí šablon můžete vytvářet podobné dokumenty - např. faktury, dopisy, faxy apod. Šablona je normální dokument, ve kterém si nadefinujete styly, rámečky, popř. text. Všechno, co se do šablony nadefinuje, se bude automaticky kopírovat do každého dokumentu otevřeného na základě této šablony.
WinText jako většina produktů od firmy Software602 obsahuje poměrně důmyslný makrojazyk, pomocí kterého můžete využívat prakticky všech funkcí textového editoru.
Součástí dodávky WinTextu je i 22 českých fontů TrueType. WinText je schopen spolupráce s dalšími aplikacemi a dále umožňuje exportovat a importovat text do mnoha různých známých formátů. Mezi podporovanými formáty nechybí ani nejnovější verze Wordu a WordPerfectu. Dalším způsobem, jak je možné používat data z jiných aplikací, je použití OLE-objektů.

WinTab
WinTab je doplňující program textového editoru WinText. Jedná se o jednoduchý tabulkový kalkulátor, pomocí kterého můžete vytvářet tabulky a zařazovat je do dokumentu a který podporuje přes 70 funkcí pro práci s tabulkami. V tabulkách můžete používat základní způsoby formátování - můžete používat různé fonty, barvy, zarovnávání apod. Pro každou buňku je možné nastavit orámování a stínování. 
Modul WinTab je chápán spíše jako doplněk textového editoru, neboť mu zcela chybí funkce pro důkladnější práci s daty (např. řešení rovnic, optimalizace, kontingentní tabulka apod.). Též mi v tomto programu chybí korektor překlepů.

WinGraf
WinGraf je modul na výrobu grafů, který navazuje na WinTab a který umožňuje vytvářet grafy různých typů. Grafy vytvořené WinGrafem  mohou být také zařazeny do textu v editoru WinText. Data se do grafů vkládají přes clipboard (schránku) a výsledné grafy jsou do textu umísťovány jako OLE-objekty.
Můžete vytvářet mnoho typů dvourozměrných i třírozměrných grafů. WinTab umožňuje pracovat se sloupcovými grafy, kruhovými grafy, páskovými grafy, X-Y grafy apod.

WinBase
WinBase je profesionální relační databáze klient/server a je na ní jasně vidět vliv Accessu a Paradoxu.  Základním oknem každé databáze (podle terminologie firmy Software602 aplikace) je základní seznam všeho, co databáze obsahuje. Přepínat se můžeme mezi seznamem tabulek, pohledů, dotazů, menu a programů.
Nejdůležitější částí databáze jsou tabulky. Jako pole tabulek můžete využívat typ řetězec, poznámka, logická hodnota, datum, čas, obrázek, OLE-objekt a různé typy čísel. U každé tabulky je také možné označit položku (položky), podle které (kterých) se bude indexovat.
Pro vybírání dat z tabulek, které splňují určitá kritéria (i poměrně složitá), slouží dotazy. Výsledkem dotazu může být nová tabulka, která obsahuje některé položky z tabulek výchozích. Další důležitou částí každé databáze jsou formuláře (pohledy). Pomocí formulářů můžete do tabulek zadávat data. 
Programovací jazyk WinBase vzdáleně vychází z Pascalu. Tento jazyk obsahuje množství funkcí, pomocí kterých můžeme programovat i poměrně složité aplikace. WinBase obsahuje také interní editor textů, ve kterém můžete programy editovat.
WinBase je opravdu zdařilý produkt, který může směle konkurovat mnoha zahraničním programům. Z jeho silných vlastností bych vyzdvihl především makrojazyk.

WinFM
Faxování “přímo z počítače” je dnes poměrně běžné a uživatel tento způsob práce ocení třeba v případě, že je třeba rozeslat jeden fax na 100 telefonních čísel anebo načasovat faxování na noc, kdy jsou poplatky za telefon nižší a linky průchodnější. Celý faxovací arzenál se skládá ze tří částí - faxového driveru, faxového serveru a prostředí WinFM602.
Faxový driver se chová jako ovladač tiskárny. Na tento ovladač je možné z libovolné aplikace ve Windows vytisknout nějaký dokument (jako na tiskárnu). Faxový driver se při “tisku” zeptá na číslo, kam má tento dokument odeslat. Potom zařadí tento fax do fronty pro faxový server. Faxový server je část vykonávající fyzickou komunikaci. Prostředí WinFM602 vám dále umožňuje prohlížet došlé a odeslané faxy nebo prohlížet frontu odeslaných a čekajících faxů.
